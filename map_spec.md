# Unfixed Earth ‚Äî Full Implementation Specification

## 1. Project Summary

**Unfixed Earth** is an interactive frontend web map visualization that lets users redefine the Earth's poles and regenerate a world map from that new spherical coordinate system.

The Earth is treated as a perfect sphere. The app does **not** rotate, pan, or re-center an existing projected map. Instead it rebuilds every lon-lat coordinate from scratch in the new frame before passing anything to a projection.

Core pipeline:

```
Earth as perfect sphere
‚Üí user clicks the 2D map to select a new north pole
‚Üí opposite point becomes new south pole
‚Üí every geometry coordinate is converted into the new coordinate system [Œª', œÜ']
‚Üí selected D3 projection is applied to the new coordinates
‚Üí new 2D world map is rendered with a blend animation
```

---

## 2. Conceptual Goal

A map is artificial ‚Äî it depends on human-defined choices: pole locations, longitude system, projection surface, map seam. This project shows that the conventional world map is only one possible mapping of the same Earth. When the user selects a new north pole the app creates a new map that is equally valid under its own coordinate system.

---

## 3. Most Important Technical Requirement

The selected point must become the actual north pole of the generated map ‚Äî not a visual center, not a pan target, not a projection center. The app must create a new coordinate system where:

```
selected point   ‚Üí new latitude +90¬∞
antipodal point  ‚Üí new latitude ‚àí90¬∞
```

**Wrong pipeline (must be avoided):**

```
Natural Earth lon-lat
‚Üí D3 projection with original coordinates
‚Üí projection.rotate() / projection.center() / CSS transform
‚Üí draw regular grid lines and call them "new coordinate system"
```

This is wrong because the original poles remain hidden anchors.

**Correct pipeline:**

```
Natural Earth lon-lat
‚Üí sphere vector v
‚Üí new coordinate system (P, E1, E2) built around selected pole
‚Üí new [Œª', œÜ'] for every point
‚Üí D3 projection receives only [Œª', œÜ']
‚Üí 2D map rendered
```

## 7. Sphere-First Coordinate Reconstruction

This is the core algorithm. It lives entirely in `src/transform.js`.

### 7.1 lon-lat ‚Üî 3D vector

```js
// degrees ‚Üí unit sphere vector
lonLatToVector(lon, lat):
  Œª = lon * œÄ/180,  œÜ = lat * œÄ/180
  return [ cos(œÜ)cos(Œª),  cos(œÜ)sin(Œª),  sin(œÜ) ]

// unit sphere vector ‚Üí degrees
vectorToLonLat(v):
  return [ atan2(v[1],v[0]) * 180/œÄ,
           asin(clamp(v[2],-1,1)) * 180/œÄ ]
```

### 7.2 Build coordinate system

`buildCoordinateSystem(poleLon, poleLat)` constructs the basis `{P, E1, E2}` around the new north pole.

```
P = lonLatToVector(poleLon, poleLat)

Reference vector A:
  if |P[0]| < 0.9  ‚Üí  A = [1,0,0]   ‚Üê chosen so that default pole (0¬∞,90¬∞)
  elif |P[1]| < 0.9 ‚Üí  A = [0,1,0]    gives the identity transformation
  else              ‚Üí  A = [0,0,1]

Gram-Schmidt onto the equatorial plane:
  Aproj = A ‚àí dot(A,P)¬∑P
  E1    = normalize(Aproj)
  E2    = cross(P, E1)          // right-hand rule; unit vector

Returns:
  { poleVector, basisE1, basisE2,
    newNorthPoleOriginalLonLat, newSouthPoleOriginalLonLat }
```

Key invariant: `buildCoordinateSystem(0, 90)` ‚Üí `E1=[1,0,0]`, `E2=[0,1,0]` ‚Üí identity transform (`A=[1,0,0]` is essential for this, not `[0,0,1]`).

### 7.3 transformPoint

Convert one original `[lon, lat]` into the new frame `[Œª', œÜ']`:

```
v      = lonLatToVector(lon, lat)
dotPV  = clamp(dot(P, v), -1, 1)
newLat = asin(dotPV) * 180/œÄ

q      = v ‚àí dotPV¬∑P              // projection onto equatorial plane
newLon = atan2(dot(q,E2), dot(q,E1)) * 180/œÄ

return [newLon, newLat]
```

### 7.4 inverseTransformPoint

Convert a redefined `[Œª', œÜ']` back to original `[lon, lat]` (used to convert a screen click to original sphere coords):

```
Œª = newLon * œÄ/180,  œÜ = newLat * œÄ/180
v = sin(œÜ)¬∑P + cos(œÜ)¬∑(cos(Œª)¬∑E1 + sin(Œª)¬∑E2)
return vectorToLonLat(v)
```

### 7.5 transformGeoJSON

Recursively transforms a GeoJSON `Feature`, `FeatureCollection`, or raw geometry by calling `transformPoint` on every coordinate. Handles Point, MultiPoint, LineString, MultiLineString, Polygon, MultiPolygon, GeometryCollection.

### 7.6 Default coordinate system

```js
const DEFAULT_SYSTEM = buildCoordinateSystem(0, 90);
```

With the default system, `transformPoint` returns the original coordinates unchanged (identity), so the map starts as a normal world map.

## 8. App State

```js
const state = {
  phase:                 'projected',   // 'projected' | 'animating'
  projType:              'equirectangular',
  graticuleMode:         'redefined',   // 'redefined' | 'original'
  customPole:            null,          // original { lon, lat }
  derivedPole:           null,          // original { lon, lat } ‚Äî antipodal
  coordinateSystem:      DEFAULT_SYSTEM,
  scrollDeg:             0,
  data:                  null,          // raw GeoJSON from loadData()
  transformedData:       null,          // { land, countries, gratOrig }
  transformedPoleCoords: null,          // { north:[Œª',œÜ'], south:[Œª',œÜ'] } | null
  cancelAnim:            null,          // cancel function for running animation
};
```

`transformedData` is rebuilt by `buildTransformedData(system)` on every pole change:

```js
function buildTransformedData(system) {
  const graticule = d3.geoGraticule()();
  return {
    land:      transformGeoJSON(state.data.land,      system),
    countries: transformGeoJSON(state.data.countries, system),
    gratOrig:  transformGeoJSON(graticule,             system),
  };
}
```

The projection **never** receives `state.data` directly ‚Äî only `state.transformedData`.

## 9. Projection Cache and Scroll

```js
const projCache = { type: null, w: null, h: null, proj: null };

function getBaseProj(type) {
  if (projCache.type !== type || projCache.w !== canvasW || projCache.h !== canvasH) {
    projCache.proj = makeProjection(type, canvasW, canvasH);
    // cache key fields updated
  }
  return projCache.proj;
}
```

Scroll is a **pure seam shift** on the already-transformed `[Œª', œÜ']` coordinates ‚Äî it is not pole simulation:

```js
function redrawProjected() {
  const proj = getBaseProj(state.projType);
  proj.rotate([state.scrollDeg, 0, 0]);  // seam shift only
  activeProjection = proj;
  draw();
}
```

`state.scrollDeg` is updated on horizontal drag/wheel (cylindrical only):

```js
function handleMapScrollDelta(pxDelta) {
  if (!isCylindrical(state.projType)) return;
  const degPerPx = 360 / Math.max(canvasW, 1);
  state.scrollDeg -= pxDelta * degPerPx;
  redrawProjected();
}
```

## 10. Pole Selection Flow

```
user clicks 2D map
‚Üí projection.invert([x,y])         ‚Üí [Œª', œÜ']  (redefined, scroll-corrected by D3)
‚Üí inverseTransformPoint(Œª', œÜ', system) ‚Üí [origLon, origLat]
‚Üí state.customPole  = { lon:origLon, lat:origLat }
‚Üí state.derivedPole = antipodal(origLon, origLat)
‚Üí state.transformedPoleCoords = {
    north: [Œª', œÜ'],              ‚Üê clicked redefined coords (immediate marker)
    south: transformPoint(derivedPole, system),
  }
‚Üí updatePoleUI()  + updateProjDiagram()
‚Üí draw()   ‚Üê show markers at clicked position
‚Üí setPhase('animating')
‚Üí setTimeout(runTransitionAnimation, 180)
```

### 10.1 runTransitionAnimation

```
oldTransformedData = state.transformedData
oldScrollDeg       = state.scrollDeg

newSystem = buildCoordinateSystem(customPole.lon, customPole.lat)
state.coordinateSystem = newSystem
state.scrollDeg        = 0
state.transformedData  = buildTransformedData(newSystem)
state.transformedPoleCoords = { north:[0,90], south:[0,-90] }
  ‚Üê in the new system the poles are always at ¬±90¬∞

oldProj = makeProjection(type, w, h);  oldProj.rotate([oldScrollDeg,0,0])
newProj = makeProjection(type, w, h);  newProj.rotate([0,0,0])

projCache invalidated

runAnim(MORPH_DURATION, d3.easeCubicInOut,
  t ‚Üí renderBlend(canvas, oldProj, newProj, oldData, newData, state, t),
  () ‚Üí finishAnimation()
)
```

### 10.2 Animation constants

```js
export const MORPH_DURATION = 1600;  // ms
```

`runAnim(duration, easing, onFrame, onDone)` uses `requestAnimationFrame` and returns a cancel function.

## 11. Rendering

All rendering is on `<canvas>` via `d3.geoPath(projection, ctx)`.

### 11.1 Colors

```js
const OCEAN_COLOR     = '#616161';
const LAND_COLOR      = '#a9b5a7';
const BORDER_COLOR    = '#232323';
const GRATICULE_COLOR = '#272727';
const GRATICULE_ORIG  = '#3a5530';
const SPHERE_STROKE   = '#282828';
const NORTH_COLOR     = '#ea4646';
const SOUTH_COLOR     = '#539eef';
```

### 11.2 renderFrame

Draw order:

1. `path(sphereFeature)` ‚Üí `fill(OCEAN_COLOR)` ‚Äî fills the projection boundary with ocean
2. `path(data.land)` ‚Üí `fill(LAND_COLOR)`
3. `drawGraticule` ‚Äî if mode is `'original'` draw `data.gratOrig` (warped); if `'redefined'` draw standard `d3.geoGraticule()()` (regular grid); lineWidth 0.5
4. `path(data.countries)` ‚Üí `stroke(BORDER_COLOR)` lineWidth 0.4
5. `path(sphereFeature)` ‚Üí `stroke(SPHERE_STROKE)` lineWidth 1
6. `drawMarkers` ‚Äî if `state.transformedPoleCoords` exists, draw N‚Ä≤ (red) and S‚Ä≤ (blue) circles + labels at the pre-transformed `[Œª', œÜ']` positions

### 11.3 renderBlend

Cross-fade from `(oldProj, oldData)` to `(newProj, newData)` at parameter `t ‚àà [0,1]`:

1. Fill sphere with ocean using `newProj`
2. `drawLayer(projA, oldData, 1-t)` ‚Äî land + redefined graticule + countries at `globalAlpha = 1-t`
3. `drawLayer(projB, newData, t)` ‚Äî same at `globalAlpha = t`
4. Stroke sphere outline with `newProj`
5. Draw pole markers using `newProj`

### 11.4 Pole markers

Drawn using pre-transformed `[Œª', œÜ']` from `state.transformedPoleCoords` (render.js has no dependency on transform.js):

```js
const p = projection([lon, lat]);   // ‚Üí [x,y] or null
// filled circle r=5.5 + label text offset 9px to the right
```

## 12. Graticule System

### Redefined graticule (default)

`d3.geoGraticule()()` is drawn **without transformation** because the projection already operates in `[Œª', œÜ']` space. The regular grid thus correctly represents the new coordinate system's parallels and meridians.

### Original graticule

`d3.geoGraticule()()` is transformed with `transformGeoJSON(graticule, system)` during `buildTransformedData()`, stored as `data.gratOrig`. In the new frame these lines appear curved/warped, showing where the original Earth grid falls.

### Toggle

Segmented control in right panel. Clicking `#grat-redefined` or `#grat-original` updates `state.graticuleMode` and redraws immediately.

`resizeCanvas(canvas)` reads `canvas.getBoundingClientRect()` and sets `canvas.width` / `canvas.height` to integer pixel dimensions. A `ResizeObserver` on the canvas calls `handleResize` which invalidates `projCache` and redraws.

## 14. Interaction

`setupInteraction(canvas, getProjection, appState, callbacks)` registers:

- `mousedown` / `mousemove` (on canvas) / `mouseup` (on window) ‚Äî click vs drag disambiguation using `DRAG_MOVED_THRESHOLD = 4 px`
- `wheel` (passive:false) ‚Äî horizontal scroll via `deltaX || deltaY`
- `touchstart` / `touchmove` / `touchend`

A click (mouse or touch) when `phase === 'projected'` calls `onPoleClick(geo.lon, geo.lat)` using `proj.invert([x,y])`.

Drag when `phase === 'projected'` calls `onMapScrollDelta(dx)` for seam shifting (horizontal only).

## 16. Dynamic Projection Diagram (`src/diagram.js`)

The left panel shows a schematic diagram that updates on every projection type change or pole selection.

### 16.1 Axis direction

The 3D pole vector is projected onto a 2D side-view plane (viewer at +X):

```js
dx = cos(lat) * sin(lon)    // east-west in diagram
dy = -sin(lat)              // up-down (negated ‚Äî north is up)
len = sqrt(dx¬≤ + dy¬≤)
if (len < 0.05) return [0,-1]   // fallback: upright (axis toward viewer)
return [dx/len, dy/len]
```

Rotation angle from vertical: `Œ∏ = atan2(adx, -ady) * 180/œÄ`

### 16.2 SVG structure

`viewBox="0 0 240 295"`, sphere at `(CX=120, CY=148, R=62)`.

All projection surface elements carry `transform="rotate(Œ∏, 120, 148)"` so the surface tilts to align with the pole axis automatically.

Z-order (SVG paint order):

```
backSurface(family)   ‚Üê drawn behind the sphere
sphere circle + decorations
frontSurface(family)  ‚Üê drawn in front of sphere
pole axis dashed line
N‚Ä≤ / S‚Ä≤ marker circles + labels
caption text
```

### 16.3 Surface geometry per family

**Cylindrical** ‚Äî cylinder with `ext=22` (extends 22px beyond sphere on each end):
- `backSurface`: bottom cap ellipse `cy=CY+R+ext, rx=R, ry=16` + body rect `fill rgba(200,220,250,0.12)`
- `frontSurface`: top cap ellipse `cy=CY-R-ext, rx=R, ry=16` (both caps `fill="#f0f0f0"`)

**Conic** ‚Äî cone tangent to sphere with `CONE_D=128, CONE_H=70`:
```
CONE_APEX = CY - 128 = 20
CONE_BASE = CY + 70  = 218
CONE_HW   = round((CONE_H+CONE_D)*R / sqrt(CONE_D¬≤-R¬≤)) ‚âà 110
```
Tangency proof: perpendicular distance from sphere centre to slant line = `CONE_D * CONE_HW / sqrt(CONE_HW¬≤ + (CONE_H+CONE_D)¬≤) = R = 62`.
- `backSurface`: base ellipse `cy=CONE_BASE, rx=CONE_HW, ry=16, fill="#f0f0f0"`
- `frontSurface`: closed triangle path `M apex L leftBase Q curveCtrl rightBase Z`, `fill rgba(200,220,250,0.15)`, then base ellipse again on top (so it appears visible through the transparent surface)

**Azimuthal** ‚Äî flat tangent plane at north pole:
- `frontSurface` only: ellipse `cy=CY-R, rx=R+20, ry=18, fill rgba(200,220,250,0.30), stroke #4a90d9`

### 16.4 Pole markers

N‚Ä≤ marker at `(CX + adx*(R+13), CY + ady*(R+13))`, S‚Ä≤ at the antipodal position. Label placed 9px to the right if marker is in the right half of the diagram, 9px to the left otherwise (`text-anchor` set accordingly).

### 16.5 When diagram updates

- `init()` ‚Äî once on startup (default upright cylindrical)
- `handleProjectionChange()` ‚Äî when dropdown changes
- `handlePoleClick()` ‚Äî after new pole is set (axis tilts)
- `handleReset()` ‚Äî reverts to default upright

## 17. Reset and Replay

**Reset** restores:
- `customPole = null`, `derivedPole = null`
- `coordinateSystem = DEFAULT_SYSTEM`
- `scrollDeg = 0`
- `graticuleMode = 'redefined'`
- `transformedPoleCoords = null`
- Rebuilds `transformedData` from `DEFAULT_SYSTEM`
- `replayBtn.disabled = true`
- Projection type **unchanged**

**Replay** re-runs the full transition animation starting from the default frame for the currently stored `customPole`. It resets `coordinateSystem` to `DEFAULT_SYSTEM` and `transformedData` to the default frame, then calls `runTransitionAnimation` after 180 ms.

After selecting a custom pole with a cylindrical projection, horizontal scroll/drag shifts `state.scrollDeg` and calls `proj.rotate([scrollDeg,0,0])`.
