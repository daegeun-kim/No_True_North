# Unfixed Earth — Full Implementation Specification

## 1. Project Summary

**Unfixed Earth** is an interactive frontend web map visualization that lets users redefine the Earth's poles and regenerate a world map from that new spherical coordinate system. It also includes a distortion analysis mode that colors each country by its size and shape distortion under the current projection.

The Earth is treated as a perfect sphere. The app does **not** rotate, pan, or re-center an existing projected map. Instead it rebuilds every lon-lat coordinate from scratch in the new frame before passing anything to a projection.

Core pipeline:

```
Earth as perfect sphere
→ user clicks the 2D map to select a new north pole
→ opposite point becomes new south pole
→ every geometry coordinate is converted into the new coordinate system [λ', φ']
→ selected D3 projection is applied to the new coordinates
→ new 2D world map is rendered with a blend animation
```

---

## 2. Conceptual Goal

A map is artificial — it depends on human-defined choices: pole locations, longitude system, projection surface, map seam. This project shows that the conventional world map is only one possible mapping of the same Earth. When the user selects a new north pole the app creates a new map that is equally valid under its own coordinate system.

---

## 3. Most Important Technical Requirement

The selected point must become the actual north pole of the generated map — not a visual center, not a pan target, not a projection center. The app must create a new coordinate system where:

```
selected point   → new latitude +90°
antipodal point  → new latitude −90°
```

**Wrong pipeline (must be avoided):**

```
Natural Earth lon-lat
→ D3 projection with original coordinates
→ projection.rotate() / projection.center() / CSS transform
→ draw regular grid lines and call them "new coordinate system"
```

This is wrong because the original poles remain hidden anchors.

**Correct pipeline:**

```
Natural Earth lon-lat
→ sphere vector v
→ new coordinate system (P, E1, E2) built around selected pole
→ new [λ', φ'] for every point
→ D3 projection receives only [λ', φ']
→ 2D map rendered
```

## 7. Sphere-First Coordinate Reconstruction

This is the core algorithm. It lives entirely in `src/transform.js`.

### 7.1 lon-lat ↔ 3D vector

```js
// degrees → unit sphere vector
lonLatToVector(lon, lat):
  λ = lon * π/180,  φ = lat * π/180
  return [ cos(φ)cos(λ),  cos(φ)sin(λ),  sin(φ) ]

// unit sphere vector → degrees
vectorToLonLat(v):
  return [ atan2(v[1],v[0]) * 180/π,
           asin(clamp(v[2],-1,1)) * 180/π ]
```

### 7.2 Build coordinate system

`buildCoordinateSystem(poleLon, poleLat)` constructs the basis `{P, E1, E2}` around the new north pole.

```
P = lonLatToVector(poleLon, poleLat)

Reference vector A:
  if |P[0]| < 0.9  →  A = [1,0,0]   ← chosen so that default pole (0°,90°)
  elif |P[1]| < 0.9 →  A = [0,1,0]    gives the identity transformation
  else              →  A = [0,0,1]

Gram-Schmidt onto the equatorial plane:
  Aproj = A − dot(A,P)·P
  E1    = normalize(Aproj)
  E2    = cross(P, E1)          // right-hand rule; unit vector

Returns:
  { poleVector, basisE1, basisE2,
    newNorthPoleOriginalLonLat, newSouthPoleOriginalLonLat }
```

Key invariant: `buildCoordinateSystem(0, 90)` → `E1=[1,0,0]`, `E2=[0,1,0]` → identity transform (A=[1,0,0] is essential for this).

### 7.3 transformPoint

Convert one original `[lon, lat]` into the new frame `[λ', φ']`:

```
v      = lonLatToVector(lon, lat)
dotPV  = clamp(dot(P, v), -1, 1)
newLat = asin(dotPV) * 180/π

q      = v − dotPV·P              // projection onto equatorial plane
newLon = atan2(dot(q,E2), dot(q,E1)) * 180/π

return [newLon, newLat]
```

### 7.4 inverseTransformPoint

Convert a redefined `[λ', φ']` back to original `[lon, lat]` (used to convert a screen click to original sphere coords):

```
λ = newLon * π/180,  φ = newLat * π/180
v = sin(φ)·P + cos(φ)·(cos(λ)·E1 + sin(λ)·E2)
return vectorToLonLat(v)
```

### 7.5 transformGeoJSON

Recursively transforms a GeoJSON `Feature`, `FeatureCollection`, or raw geometry by calling `transformPoint` on every coordinate. Handles Point, MultiPoint, LineString, MultiLineString, Polygon, MultiPolygon, GeometryCollection.

### 7.6 Default coordinate system

```js
const DEFAULT_SYSTEM = buildCoordinateSystem(0, 90);
```

With the default system, `transformPoint` returns the original coordinates unchanged (identity), so the map starts as a normal world map.

`transformedData` is rebuilt by `buildTransformedData(system)` on every pole change:

```js
function buildTransformedData(system) {
  const graticule = d3.geoGraticule()();
  return {
    land:      transformGeoJSON(state.data.land,      system),
    countries: transformGeoJSON(state.data.countries, system),
    gratOrig:  transformGeoJSON(graticule,             system),
  };
}
```

The projection **never** receives `state.data` directly — only `state.transformedData`.

## 9. Projection Cache and Scroll

```js
const projCache = { type: null, w: null, h: null, proj: null };

function getBaseProj(type) {
  if (projCache.type !== type || projCache.w !== canvasW || projCache.h !== canvasH) {
    projCache.proj = makeProjection(type, canvasW, canvasH);
    // cache key fields updated
  }
  return projCache.proj;
}
```

Scroll is a **pure seam shift** on the already-transformed `[λ', φ']` coordinates — it is not pole simulation:

```js
function redrawProjected() {
  const proj = getBaseProj(state.projType);
  proj.rotate([state.scrollDeg, 0, 0]);  // seam shift only
  activeProjection = proj;
  draw();
}
```

`state.scrollDeg` is updated on horizontal drag/wheel (cylindrical only):

```js
function handleMapScrollDelta(pxDelta) {
  if (!isCylindrical(state.projType)) return;
  const degPerPx = 360 / Math.max(canvasW, 1);
  state.scrollDeg -= pxDelta * degPerPx;
  redrawProjected();
}
```

## 10. Pole Selection Flow

```
user clicks 2D map
→ projection.invert([x,y])             → [λ', φ']  (redefined, scroll-corrected by D3)
→ inverseTransformPoint(λ', φ', system) → [origLon, origLat]
→ state.customPole  = { lon:origLon, lat:origLat }
→ state.derivedPole = antipodal(origLon, origLat)
→ state.transformedPoleCoords = {
    north: [λ', φ'],                   ← clicked redefined coords (immediate marker)
    south: transformPoint(derivedPole, system),
  }
→ updatePoleUI()  + updateProjDiagram()
→ draw()   ← show markers at clicked position
→ setPhase('animating')
→ setTimeout(runTransitionAnimation, 180)
```

### 10.1 runTransitionAnimation

```
oldTransformedData = state.transformedData
oldScrollDeg       = state.scrollDeg

newSystem = buildCoordinateSystem(customPole.lon, customPole.lat)
state.coordinateSystem = newSystem
state.scrollDeg        = 0
state.transformedData  = buildTransformedData(newSystem)
state.transformedPoleCoords = { north:[0,90], south:[0,-90] }
  ← in the new system the poles are always at ±90°

oldProj = makeProjection(type, w, h);  oldProj.rotate([oldScrollDeg,0,0])
newProj = makeProjection(type, w, h);  newProj.rotate([0,0,0])

projCache invalidated (projCache.type = null)

runAnim(MORPH_DURATION, d3.easeCubicInOut,
  t → renderBlend(canvas, oldProj, newProj, oldData, newData, state, t),
  () → finishAnimation()
)
```

### 10.2 finishAnimation

```
activeProjection = newProj (retrieved via getBaseProj)
setPhase('projected')
replayBtn.disabled = false
setStatus('')

if state.distortionMode:
  state.distortionMap = null
  computeDistortionMap()      ← recomputes and redraws with distortion colors
else:
  draw()
```

### 10.3 Animation constants

```js
export const MORPH_DURATION = 1600;  // ms  (used by main.js)
export const ROTATION_DURATION = 1200;  // ms  (exported but unused)
```

`runAnim(duration, easing, onFrame, onDone)` uses `requestAnimationFrame` and returns a cancel function.

### 11.2 renderFrame

`renderFrame(canvas, proj, data, state, distOpts)`

`distOpts` is `null` in normal mode or `{ active, distortionMap, sizeColor, shapeColor }` when distortion is on.

Draw order:

1. `path(sphereFeature)` → `fill(OCEAN_COLOR)` — fills the projection boundary with ocean
2. `path(data.land)` → `fill(distOpts?.active ? DISTORTION_COLOR_CONFIG.noDataColor : LAND_COLOR)` — in distortion mode, land gets `'#d4d4d4'` as neutral base
3. If `distOpts.active && distortionMap.size > 0`: call `drawCountryDistortionLayer` — fills each country polygon with its bivariate color at `globalAlpha = DISTORTION_COLOR_CONFIG.countryOpacity` (0.78)
4. `drawGraticule` — if mode is `'original'` draw `data.gratOrig` (warped); if `'redefined'` draw standard `d3.geoGraticule()()` (regular grid); `lineWidth 0.5`
5. `path(data.countries)` → `stroke(BORDER_COLOR)` `lineWidth 0.4`
6. `path(sphereFeature)` → `stroke(SPHERE_STROKE)` `lineWidth 1`
7. `drawMarkers` — if `state.transformedPoleCoords` exists, draw N′ (red) and S′ (blue) circles + labels

### 11.3 renderBlend

`renderBlend(canvas, projA, projB, oldData, newData, state, t)`

Cross-fade from `(projA, oldData)` to `(projB, newData)` at parameter `t ∈ [0,1]`. No distortion coloring during blend.

1. Fill sphere with ocean using `projB`
2. `drawLayer(projA, oldData, 1-t)` — land (LAND_COLOR) + redefined graticule + countries stroke, `globalAlpha = 1-t`
3. `drawLayer(projB, newData, t)` — same at `globalAlpha = t`
4. Reset `globalAlpha = 1`, stroke sphere outline with `projB`
5. Draw pole markers using `projB`

### 11.4 Pole markers

Drawn using pre-transformed `[λ', φ']` from `state.transformedPoleCoords` (render.js has no dependency on transform.js):

```js
const p = projection([lon, lat]);   // → [x,y] or null
```

## 12. Distortion Analysis System

Lives in `src/distortion.js`. Implements country-level bivariate distortion using Tissot-style differential approximation. Distortion is measured relative to the projection's own scale distribution — the default map also shows distortion.

### 12.1 Color configuration

Exported at the top of `distortion.js` for easy tuning:

```js
export const DISTORTION_COLOR_CONFIG = {
  sizeColor:      '#2563eb',   // reference only — actual colors come from state
  shapeColor:     '#dc2626',   // reference only
  lowColor:       '#f8fafc',
  borderColor:    '#111827',
  countryOpacity: 0.78,        // used by drawCountryDistortionLayer
  noDataColor:    '#d4d4d4',   // used as land base color in distortion mode
};
```

### 12.2 Hex/RGB conversion

```js
export function hexToRgb(hex)
  // strips '#', parses 6-digit hex → [r, g, b]
```

### 12.3 Bivariate color interpolation

`bivariateColor(s, q, sizeHex, shapeHex)` — bilinear interpolation across four corners:

```
corners:
  (s=0, q=0) → neutral light [228, 228, 228]
  (s=1, q=0) → sizeColor
  (s=0, q=1) → shapeColor
  (s=1, q=1) → average(sizeColor, shapeColor)

r = (1-s)(1-q)·wr + s(1-q)·sr + (1-s)q·qr + sq·mr
(same formula for g, b)

returns [r, g, b] clamped to [0, 255]
```

### 12.4 Tissot h/k scale factors

`computeHK(projection, lp, fp)` — finite-difference Jacobian using `EPS = 0.35°`:

```
pN = projection([lp, fp + EPS])  (clamped to ≤89.5°)
pS = projection([lp, fp - EPS])  (clamped to ≥-89.5°)
pE = projection([lp + EPS, fp])
pW = projection([lp - EPS, fp])

h = |pN - pS| / (fpN - fpS)         ← meridional scale
k = |pE - pW| / ((lpE - lpW) · cos(fp)) ← parallel scale

returns { h, k } or null if any projected point is NaN / out of range
```

`lp` and `fp` are coordinates in the redefined `[λ', φ']` system (what D3 projection receives).

### 12.5 Country sample point precomputation

`precomputeCountrySamples(countriesGeoJSON)` — called once after data load, returns `Map<id, [[lon,lat],...]>` in **original** geographic coordinates.

Sample count per country based on `d3.geoArea(feature)`:
- area > 0.01 → 20 points
- area > 0.001 → 8 points
- else → 2 points

Sampling: grid over bounding box, `d3.geoContains` check. Countries spanning the antimeridian (`lon1 - lon0 > 170`) fall back to centroid. Falls back to `d3.geoCentroid` if no points found.

### 12.6 Country distortion computation

`computeCountryDistortions(originalCountries, countrySamples, projection, coordinateSystem)`

Returns `Map<id, {sizeNorm, shapeNorm}>` with values in [0, 1].

**First pass** — collect h/k at every sample point across all countries:

```
for each feature:
  for each [lon, lat] in countrySamples.get(feature.id):
    [lp, fp] = transformPoint(lon, lat, coordinateSystem)
    hk = computeHK(projection, lp, fp)
    if hk: store hk for this country; push hk.h * hk.k to global allHK array

medianHK = median(allHK)   ← global reference for size
```

**Second pass** — per-country raw distortion values:

```
for each country:
  avgHK    = mean(h·k across sample points)
  sizeRaw  = |log(avgHK / medianHK)|
  shapeRaw = mean( |log(max(h,k) / min(h,k))| ) across sample points
```

`sizeRaw` uses global median as reference so the distortion reflects each country's deviation from the projection's typical area scale (not from a perfectly equal-area sphere projection). `shapeRaw` is the average log-eccentricity of the Tissot ellipse (0 = locally conformal).

**Normalization** — 95th-percentile clamping to prevent outliers from dominating:

```
size95  = percentile(all sizeRaw values, 0.95)
shape95 = percentile(all shapeRaw values, 0.95)
sizeNorm  = min(1, sizeRaw / size95)
shapeNorm = min(1, shapeRaw / shape95)
```

Countries with no valid sample points get `{ sizeNorm: 0, shapeNorm: 0 }`.

### 12.7 Distortion preset search

`searchDistortionPreset(key, originalCountries, makeProjectionFn, width, height)`

`key` is `'minSize' | 'maxSize' | 'minShape' | 'maxShape'`. Returns `[lon, lat]` of the pole that produces the best global distortion score.

Uses only country centroids (not full sample grids) for speed.

**Coarse search** — 30° × 30° grid (lon −150→180, lat −75→90):

```
for each (lon, lat) candidate:
  system = buildCoordinateSystem(lon, lat)
  proj   = makeProjectionFn(width, height); proj.rotate([0,0,0])
  score  = area-weighted mean distortion across all centroids
```

**Fine search** — ±20° around best candidate, 5° step.

`scoreCandidate` computes global medianHK from all centroids, then for each centroid:

```
score_size  = |log(hk.h·hk.k / medianHK)|
score_shape = |log(max(h,k) / min(h,k))|
weighted by d3.geoArea(feature)
```

## 13. Graticule System

### Redefined graticule (default)

`d3.geoGraticule()()` is drawn **without transformation** because the projection already operates in `[λ', φ']` space. The regular grid thus correctly represents the new coordinate system's parallels and meridians.

### Original graticule

`d3.geoGraticule()()` is transformed with `transformGeoJSON(graticule, system)` during `buildTransformedData()`, stored as `data.gratOrig`. In the new frame these lines appear curved/warped, showing where the original Earth grid falls.

### Toggle

Segmented control in right panel. Clicking `#grat-redefined` or `#grat-original` updates `state.graticuleMode` and redraws immediately. Both buttons reference `GRATICULE_COLOR` / `GRATICULE_ORIG` (both `'#434343'` in current code).

## 14. Canvas Sizing

`resizeCanvas(canvas)` reads `canvas.getBoundingClientRect()` and sets `canvas.width` / `canvas.height` to integer pixel dimensions. A `ResizeObserver` on the canvas calls `handleResize` which invalidates `projCache` and calls `redrawProjected()`.

## 15. Interaction

`setupInteraction(canvas, getProjection, appState, callbacks)` — returns a `teardown()` function.

Constants:
```js
const GLOBE_SENSITIVITY    = 0.3;  // degrees per CSS pixel (for globe drag — unused in current app)
const DRAG_MOVED_THRESHOLD = 4;    // px — below this, mouseup counts as a click
```

Registered events:
- `mousedown` on canvas — record start position
- `mousemove` on canvas — dispatch drag if moved > threshold
- `mouseup` on window — on click: `onPoleClick(geo.lon, geo.lat)`
- `wheel` on canvas (passive:false) — `onMapScrollDelta(deltaX || deltaY)`
- `touchstart` / `touchmove` / `touchend` on canvas

A click (mouse or touch) when `phase === 'projected'` calls `onPoleClick` using `proj.invert([x,y])`.

Drag when `phase === 'projected'` calls `onMapScrollDelta(dx)` for horizontal seam shifting.

Canvas coordinates are DPI-corrected: `(clientX - rect.left) * (canvas.width / rect.width)`.

## 17. Dynamic Projection Diagram (`src/diagram.js`)

The left panel shows a schematic diagram that updates on every projection type change or pole selection.

### 17.1 Axis direction

The 3D pole vector is projected onto a 2D side-view plane (viewer at +X):

```js
dx = cos(lat) * sin(lon)    // east-west in diagram
dy = -sin(lat)              // up-down (negated — north is up)
len = sqrt(dx² + dy²)
if (len < 0.05) return [0,-1]   // fallback: upright (axis toward viewer)
return [dx/len, dy/len]
```

Rotation angle from vertical: `θ = atan2(adx, -ady) * 180/π`

### 17.2 SVG structure

`viewBox="0 0 240 295"`, sphere at `(CX=120, CY=148, R=62)`.

All projection surface elements carry `transform="rotate(θ, 120, 148)"` so the surface tilts to align with the pole axis automatically.

Z-order (SVG paint order):

```
backSurface(family)   ← drawn behind the sphere
sphere circle + decorations (equatorial ellipse, meridian ellipse)
frontSurface(family)  ← drawn in front of sphere
pole axis dashed line (N to S through sphere)
N′ / S′ marker circles + labels
caption text
```

Globe sphere: `fill="#e8ede8" stroke="#666"`. Ellipses: `stroke="#c0c0c0" stroke-dasharray="5,3"`.

### 17.3 Surface geometry per family

**Cylindrical** — `ext=22` (extends 22px beyond sphere on each end):
- `backSurface`: bottom cap ellipse `cy=CY+R+ext, rx=R, ry=16, fill="#f0f0f0"` + body rect `fill rgba(200,220,250,0.12) stroke #aaa`
- `frontSurface`: top cap ellipse `cy=CY-R-ext, rx=R, ry=16, fill="#f0f0f0"`

**Conic** — `CONE_D=128, CONE_H=70, CONE_APEX=CY-128=20, CONE_BASE=CY+70=218`:
```
CONE_HW = round((CONE_H+CONE_D) * R / sqrt(CONE_D²-R²)) ≈ 110
```
- `backSurface`: base ellipse `cy=CONE_BASE, rx=CONE_HW, ry=16, fill="#f0f0f0"`
- `frontSurface`: transparent triangle path apex→left/right base with quadratic curve at bottom `fill rgba(200,220,250,0.15)`, then base ellipse again on top

**Azimuthal** — flat tangent plane at north pole:
- `frontSurface` only: ellipse `cy=CY-R, rx=R+20, ry=18, fill rgba(200,220,250,0.30), stroke #4a90d9 stroke-width 1.4`

### 17.4 Pole markers

N′ at `(CX + adx·(R+13), CY + ady·(R+13))`, r=6, fill `#ea4646`.
S′ at `(CX - adx·(R+13), CY - ady·(R+13))`, r=6, fill `#539eef`.
Label placed 9px to the right/left based on which half of the diagram the marker falls in (`text-anchor: start/end`).

### 17.5 Caption text

```js
{
  cylindrical: 'Cylinder unwraps to flat rectangle',
  conic:       'Cone unwraps to fan shape',
  azimuthal:   'Projected onto flat plane',
}
```

Rendered at `x=120, y=288, font-size=11, fill=#888`.

### 17.6 When diagram updates

- `init()` — once on startup (default upright cylindrical)
- `handleProjectionChange()` — when dropdown changes
- `handlePoleClick()` — after new pole is set (axis tilts)
- `handleDistortionPreset()` — after preset-driven pole change
- `handleReset()` — reverts to default upright

## 18. Reset and Replay

**Reset** restores:
- `customPole = null`, `derivedPole = null`
- `coordinateSystem = DEFAULT_SYSTEM`
- `scrollDeg = 0`
- `graticuleMode = 'redefined'`
- `transformedPoleCoords = null`
- `distortionMap = null`
- Rebuilds `transformedData` from `DEFAULT_SYSTEM`
- `replayBtn.disabled = true`
- Projection type and `distortionMode` **unchanged**
- If `distortionMode` is on, calls `computeDistortionMap()` after reset

**Replay** re-runs the full transition animation starting from the default frame for the currently stored `customPole`. It resets `coordinateSystem` to `DEFAULT_SYSTEM` and `transformedData` to the default frame, then calls `runTransitionAnimation` after 180 ms.

## 22. Validation Tests

### 22.1 Identity test (default pole)

`buildCoordinateSystem(0, 90)` must produce `E1=[1,0,0]`, `E2=[0,1,0]`.
`transformPoint(λ, φ, DEFAULT_SYSTEM)` must return `[λ, φ]` for any input.

### 22.2 Pole self-test

```
transformPoint(30, 45, buildCoordinateSystem(30, 45))
→ latitude ≈ +90°
```

### 22.3 Antipodal test

```
transformPoint(-150, -45, buildCoordinateSystem(30, 45))
→ latitude ≈ −90°
```

### 22.4 Cylindrical appearance

After selecting `[30°, 45°]` as new north pole with Equirectangular:
- `[30°, 45°]` appears on the top edge of the map.
- `[-150°, -45°]` appears on the bottom edge.
- Original north pole is an ordinary interior point.
- Redefined graticule appears as normal horizontal/vertical grid.
- Original graticule appears curved.

### 22.5 Scroll test

After selecting a custom pole with a cylindrical projection, horizontal scroll/drag shifts `state.scrollDeg` and calls `proj.rotate([scrollDeg,0,0])`. The seam shifts but the pole axis remains the map's top/bottom anchor. The original north pole does **not** return to the top edge during scrolling.

### 22.6 Replay test

After replay, the animation starts from the default map and morphs to the same custom-pole map as after the original click.

### 22.7 Distortion mode test

- On first click of "Show Distortion Compare": `computeDistortionMap` runs, `state.distortionMap` is populated, every country polygon gets a bivariate fill color.
- Default map (no custom pole, equirectangular) should show non-zero distortion values.
- "Hide" then "Show" again: `state.distortionMap` is still set, `draw()` is called directly without recomputing.
- After a pole change or projection change: `state.distortionMap` is reset to `null` and recomputed.
- Legend canvas shows a bivariate color gradient matching `state.sizeColor` and `state.shapeColor`.

## 23. Known Workarounds

| Projection key | D3 constructor actually used | Reason |
|---|---|---|
| `lambertConformalConic` | `d3.geoConicEquidistant().parallels([40,30])` | `geoConicConformal` with `fitExtent({ type:'Sphere' })` produces degenerate scale for full-world display |
| `lambertAzimuthalEqualArea` | `d3.geoConicEquidistant().parallels([90,90])` | `geoAzimuthalEqualArea` with `clipAngle(180)` + `fitExtent` produces scale=0 in some environments |

The UI labels remain "Lambert Conformal Conic" and "Lambert Azimuthal Equal-Area" — only the underlying D3 constructor is substituted.
